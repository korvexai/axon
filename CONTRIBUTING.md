# Contributing to AXON

Thank you for your interest in contributing to AXON.

AXON is an independent, AGPLv3-licensed project focused on local-first development infrastructure written in Rust.

---

## Before Contributing

Please ensure:

- Code compiles without warnings.
- `cargo fmt` passes.
- `cargo clippy` passes.
- Logic follows the event-driven architecture.
- Changes are modular and documented.

---

## Code Standards

- All logic must be modular.
- Architecture must remain event-driven.
- Avoid global mutable state.
- Respect worker isolation principles.
- Prefer explicit over implicit behavior.

---

## Pull Request Process

1. Fork the repository.
2. Create a feature branch.
3. Ensure formatting and lint checks pass.
4. Submit a Pull Request with a clear description.

---

## License Agreement

By submitting a Pull Request to this repository, you agree to the terms outlined in the Contributor License Agreement (CLA).

See `CLA.md` for full details.

---

## Philosophy

AXON prioritizes:

- Transparency
- Deterministic behavior
- Local-first infrastructure
- System-level orchestration

Contributions should respect these principles.

---

Thank you for contributing.
